## CLAUDE.md テンプレート

### 秘書室 (Secretary Office)
```markdown
# 秘書室 (Secretary Office)

## 役割・責務
秘書室は経営陣のサポート、情報の一元管理、全社調整を担う部門です。CEO直下に位置し、以下の業務を統括します：
- スケジュール管理・アポイント調整
- 重要な意思決定のサポート（壁打ち、メモ化）
- 全社への重要通知・連絡
- 情報資産の一元化と共有
- 部門間の連携調整

## フォルダ構成
```
secretary/
├── inbox/          # 受信・相談事項の管理
├── todos/          # CEO・経営陣の優先業務
├── notes/          # 重要議議事録・メモ
```
```

## 基本ルール
1. **ファイル命名**
   - `inbox_YYYYMMDD.md` - 日単位の受信管理
   - `todos_YYYY-MM.md` - 月単位のタスク管理
   - `notes_テーマ_YYYYMMDD.md` - テーマ別メモ

2. **情報の機密性・優先度付け**
   - 経営レベルの情報は厳格に管理
   - 各部門からの相談は対応順序を記録
   - 完了後は完了日時を記載

3. **部門間連携**
   - 全社に関わる重要事項は秘書室から各部門に通知
   - 部門の課題や調整が必要な場合は当事者部門に連絡
   - 月1回の全体調整会議の議事録を保管

## クロスファンクション連携
- **全部門**: 重要通知・期限・重要事項の共有
- **組織開発部**: 研修企画・人事課題相談
- **マーケ戦略部**: 経営方針・四半期目標の策定
```

```markdown
# 総務部 (General Affairs)

## 役割・責務
総務部は組織全体の運営、管理業務、サポート機能を担当する部門です。以下を統括します：
- 日常業務管理・スケジュール調整
- 経費管理・承認フロー
- 契約管理・法務支援
- 施設・環境管理
- 全社的な管理業務の企画・実行

## フォルダ構成
```
general-affairs/
├── tasks/          # 定期・臨時業務の管理
```
```

## 基本ルール
1. **ファイル命名**
   - `tasks_YYYY-MM.md` - 月単位の業務リスト

2. **業務管理**
   - 定期業務（毎月実施）と臨時業務を分類
   - 期限・担当・優先度を明記
   - 完了後は完了日を記録

3. **部門間連携**
   - 全部門の業務がスムーズに進むようサポート
   - 経費申請や契約に関する質問は速やかに対応
   - 月次報告は秘書室へ提出

## クロスファンクション連携
- **秘書室**: 重要な管理課題・経営判断が必要な事項を報告
- **全部門**: 経費精算・契約・施設利用の支援
```

## 全社共通ルール

### 部門横断的ルール
1. **月次報告** - 各部門長は月末に秘書室に活動報告を提出
2. **相互協力** - 他部門の依頼・調整は優先度を考慮して対応
3. **情報共有** - 全社に影響する情報は秘書室経由で通知
4. **ドキュメント管理** - 全社で統一したフォーマット・命名規則を遵守
5. **品質基準** - ブランドガイドラインに準拠した全てのコンテンツ・資料

### 各部門連携マップ
- **秘書室** ↔ 全部門（調整・報告）
- **マーケ戦略部** → 全マーケ部門（目標指示・方針）
- **データ分析部** ← 全部門（データ提供・分析依頼）
- **ブランディング部** → 全部門（ガイドライン遵守）
- **コンテンツ制作部** → SNS運用部・メルマガ部・イベント部（素材提供）
