# CS部 (Customer Success)

## 役割・責務

CS部は顧客満足度向上、長期的な顧客関係構築、リテンション拡大を推進する部門です：
- 顧客のオンボーディング・成功支援
- 定期的なチェックイン・ヘルスチェック
- 顧客フィードバック・要望収集
- 顧客成功事例の発掘・共有
- 顧客満足度向上施策の企画

## フォルダ構成

```
cs/
├── projects/               # CSプロジェクト
│   ├── kintai/             勤労の獅子
│   └── hr-consulting/      HRコンサルティング
└── feedback/               # 顧客フィードバック
    ├── kintai/             勤労の獅子
    └── hr-consulting/      HRコンサルティング
```

## サービス別フォルダの使い方

- `projects/kintai/` : 勤怠管理導入顧客の支援記録・成功事例・活用Tips
- `projects/hr-consulting/` : HRコンサル顧客のプロジェクト進捗・成果報告
- `feedback/kintai/` `feedback/hr-consulting/` : サービス別の顧客フィードバック集約
- ファイル命名: `顧客名_YYYY.md` または `YYYYMM_事例タイトル.md`

## 基本ルール

1. **ファイル命名**
   - `projects_顧客名_YYYY.md` - 顧客別プロジェクト
   - `feedback_YYYY-MM.md` - 月単位のフィードバック集約

2. **顧客管理**
   - 顧客の成功指標（導入効果測定）を明確化
   - 定期チェックイン・QBRで関係強化
   - フィードバック・要望は構造化して記録
   - 改善提案を各部門に連携

3. **フィードバック管理**
   - 顧客要望・不具合報告を月次集約
   - 要望の優先度（要件定義チームへの提案優先度）を判定
   - 頻出課題は組織開発部と共有（研修テーマ化）

## 顧客セグメント別戦略

**導入初期顧客**
- 定期チェックイン（週1回）
- 運用ガイダンス・トレーニング
- ROI達成支援

**既存顧客**
- 月次定期会議
- 利用状況分析・最適化提案
- アップセル・クロスセル機会識別

**大口顧客**
- 四半期ビジネスレビュー（QBR）
- 経営層との関係構築
- カスタマイズ・専用ソリューション検討

**チャーンリスク顧客**
- 集中的なフォロー
- ニーズ再分析
- 利用価値の再提示

## KPI管理

- 顧客満足度（CSAT）: 80%以上
- 顧客推奨度（NPS）: 40以上
- チャーン率: 5%以下（年間）
- 展開サービス数: 平均2.5サービス以上/顧客

## クロスファンクション連携

- **秘書室**: 大口顧客対応・エスカレーション
- **コンテンツ制作部**: 成功事例・顧客事例の活用
- **イベント企画部**: ユーザー会参加者・登壇者の紹介
- **メルマガ部**: 顧客セグメント・成功事例紹介
- **データ分析部**: 顧客満足度・解約率等の分析
